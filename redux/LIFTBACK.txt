SpliceLiftBack code analysis
============================

Purpose
-------

After lifting back BWA-MEM2 produced BAM, the output should look like an ordinary genomic RNA BAM:

  - no *_tx contigs in the output header
  - no *_tx coordinates on records
  - XA and SA tags rewritten to genomic coordinates
  - mate fields patched to genomic coordinates
  - spliced reads represented with N CIGAR operations

Where it fits in the splice prototype
-------------------------------------
The overall flow is:

  1. SpliceFastaBuilder builds transcript contigs from annotated ensembl multi-exon transcripts and writes a sidecar
     TSV that maps each packed transcript interval back to gene, transcript, chromosome, and exon spans.
  2. The generated transcript FASTA is appended to the normal genome FASTA.
  3. bwa-mem2 indexes and aligns RNA reads against that combined FASTA.
  4. SpliceLiftBack reads the bwa-mem2 BAM and sidecar, then rewrites tx-contig alignments back to the genome.

This is trying to recover much of STAR's splice-aware behavior while using bwa-mem2 as the aligner.

Required inputs
---------------
SpliceLiftBackConfig requires:

  -input_bam
      bwa-mem2 BAM aligned against the combined genome + transcript FASTA.

  -ref_genome
      The same combined FASTA used during bwa-mem2 alignment. This matters because htsjdk reads the input BAM
      against that dictionary, and rescue and tail extension use this FASTA for reference-base lookups.

  -ref_genome_version
      Standard hmftools reference selector, usually V38 for this prototype.

  -contig_sidecar
      The *.rna_contigs_mappings.tsv from SpliceFastaBuilder. This is the critical lookup table for
      translating packed *_tx coordinates back to genomic exon spans.

  -ensembl_data_dir
      Ensembl data cache. Used for annotated exons, annotated junctions, MAPQ hidden-tie handling, rescue, and
      softclip-tail guards.

  -output_dir
      Destination directory. Created if missing.

Useful optional inputs:

  -output_id
      Prefix for output files. Default is splice_lifted.

  -bamtool
      samtools path. Used for sorting/indexing, and required by -emit_secondaries because that mode does a
      name-sort pre-pass.

  -threads
      Passed to samtools sort/index/name-sort.

  -emit_secondaries
      Use when bwa-mem2 was run with -a and alternate placements arrive as 0x100 secondary records. Without
      this, alternate placements are expected mainly in XA tags.

  -rescue_via_supp
      Try to merge primary + supplementary pieces into one spliced primary.

  -extend_softclip_tails
      Recover terminal softclipped bases that match the reference.

  -unmap_above_nh / -unmap_below_mapq
      Optional primary-only filters after lift-back.

Outputs
-------
For output_id SAMPLE:

  SAMPLE.bam
      Sorted lifted genomic BAM, when -bamtool is supplied.

  SAMPLE.bam.bai
      BAM index, when -bamtool is samtools.

  SAMPLE.unsorted.bam
      Temporary unsorted BAM. Deleted after sort. If no -bamtool is supplied, this is left as the usable BAM
      output.

  SAMPLE.liftback.records.tsv
      One row per emitted record. This is the main debug file. It says the final category, final coordinates,
      input/final MAPQ, NumLoci, gene IDs, and notes such as rescued-via-supp or tail-extended.

  SAMPLE.liftback.alignments.tsv
      One row per lifted candidate alignment considered for each emitted row. This is the "what evidence did
      the resolver see?" file.

  SAMPLE.liftback.summary.tsv
      Run-level counters by alignment composition, decision category, MAPQ tier, and primary bucket.

Main code map
-------------
The core classes are:

  SpliceLiftBack.java
      Main orchestration: load inputs, build mate cache, stream records, apply rescue/extension, mutate
      SAMRecords, write output, sort/index.

  SpliceLiftBackConfig.java
      Command-line flags and output file names.

  ContigSidecar.java / ContigEntry.java
      Read the sidecar mapping from packed tx-contig intervals to transcript exon spans.

  ContigTranslator.java
      Translate a tx-contig CIGAR into a genomic CIGAR with N introns.

  LiftBackResolver.java
      Resolve one SAMRecord into a LiftBackResult. It lifts SELF plus XA or secondary candidates, runs the
      discriminator, and applies MAPQ rules.

  LiftBackDiscriminator.java / LiftBackCategory.java
      Decide whether ref evidence, tx evidence, or neither clearly wins.

  LiftBackWriter.java / LiftBackStats.java
      TSV debug output and run summaries.

  rescue/JunctionRescueResolver.java
      Optional primary + supplementary merge into a spliced primary.

  tailextend/SoftclipTailExtender.java
      Optional terminal softclip cleanup.

Top-level runtime flow
----------------------
SpliceLiftBack.run() does these major steps:

  1. Optional name-sort
     If -emit_secondaries is set, nameSortInput() runs samtools sort -n so all records for a read name are
     contiguous. This lets the resolver see the primary and its 0x100 secondary alternates together.

  2. Load lookup data
     ContigSidecar.read() loads tx-contig mappings. ExonRegionIndex.load() loads annotated exon regions. If
     rescue or tail extension is enabled, AnnotatedJunctionLoader also loads annotated introns.

  3. Validate BAM/header against sidecar
     validateBamAgainstSidecar() checks that every *_tx contig in the BAM header exists in the sidecar.
     Missing sidecar entries are a hard error, because otherwise tx contigs could pass through as fake
     reference contigs.

  4. Pass 1: lifted mate cache
     buildLiftedMateInfoCache() scans primary paired records, resolves their lifted positions, and stores
     lifted mate information. Pass 2 uses this to patch mate ref, mate position, mate strand, MC, TLEN, and
     proper-pair state after the records are rewritten.

  5. Pass 2: resolve and write
     In normal mode, streamPerRecord() resolves one record at a time. In -emit_secondaries mode,
     streamGrouped() buffers read-name groups, splits mate 1 and mate 2, resolves the primary plus secondaries
     as a group, then writes/dedups the mate group.

  6. Sort/index
     sortAndIndex() sorts the unsorted BAM and indexes it if the configured tool is samtools.

How tx-coordinate translation works
-----------------------------------
LiftBackResolver.liftAlignment() is the core lift function.

If the source contig is not in the sidecar:

  - normal ref contigs pass through unchanged
  - names ending in *_tx fail, because that indicates a FASTA/sidecar mismatch

If the source contig is a known tx contig:

  1. findSegment() finds the ContigEntry whose packed alt-contig interval owns the alignment start. If the
     start lands just into N spacer between packed transcripts, it chooses the closest neighboring segment and
     lets the translator clamp the overhang into softclip.

  2. ContigTranslator.translate() walks the CIGAR through the transcript's exon spans. Reference-consuming
     operations are copied until an exon boundary is crossed, then an N is inserted for the intron and walking
     continues in the next exon.

  3. Small D operations next to a newly emitted N can be absorbed into the N. This handles tiny boundary
     offsets without representing them as real deletions.

  4. trimMicroAnchors() removes tiny terminal splice anchors under 3 bp by folding them into softclip. This
     avoids treating a few accidental bases across an exon edge as real splice support.

The LiftedAlignment records both original and lifted coordinates:

  OrigContig / OrigPos / OrigCigar
  LiftedChrom / LiftedPos / LiftedCigar
  transcript and gene metadata if it came from a tx contig
  whether the tx alignment softclipped at an exon boundary

Candidate alignments
--------------------
For a primary record, LiftBackResolver builds a candidate set:

  SELF
      The record's own alignment.

  XA_INPUT
      XA tag alternates, in normal mode.

  SECONDARY_INPUT
      0x100 secondary records, in -emit_secondaries mode.

XA entries are parsed and lifted, then deduplicated by lifted chrom/pos/CIGAR. SELF is not deduplicated
against XA because ref and tx agreeing at one locus is important evidence.

Supplementary records are resolved independently. They are split-read pieces, not alternate placements, so
they do not participate in the primary's ref-vs-tx discriminator set.

Decision categories
-------------------
LiftBackDiscriminator categorizes the candidate set.

Single-source categories:

  REF_SINGLE / TX_SINGLE
      One genomic locus from only ref or only tx evidence.

  REF_MULTI / TX_MULTI
      Multiple loci from one source class. These are problem/multi categories.

Ref + tx categories:

  BOTH_AGREE
      Ref and tx land at the same locus with the same intronless CIGAR.

  BOTH_TX_JUNCTION_REF_SOFTCLIP
      Tx has a real N junction and ref softclips around it. Tx wins.

  BOTH_TX_JUNCTION_REF_MATCH
      Tx has an N, but ref fully matches through the supposed intron. Ref wins; this looks like retained
      intron, pre-mRNA, or DNA contamination rather than a spliced read.

  BOTH_TX_SOFTCLIP_REF_MATCH
      Tx softclips at an exon edge while ref matches cleanly. Ref wins.

  BOTH_MULTI_TX_JUNCTION
      Tx has splice evidence and ref alternatives hit other intronless loci. Tx wins; this is often a
      processed pseudogene/paralog pattern.

  BOTH_MULTI / BOTH_AMBIGUOUS
      No confident rule resolves the evidence.

PrimaryBucket in records.tsv is a coarse view:

  RESOLVED
      Confident single-locus or confident ref-vs-tx decision.

  PROBLEM
      Multi/ambiguous primary categories.

  NON_PRIMARY
      Unmapped, lift failed, or supplementary/secondary output.

What winning changes
--------------------
The discriminator can mark losing alignments Dropped and can change which candidate is the effective primary.

In normal XA mode, if a better XA candidate wins, the BAM primary is rewritten to that candidate's lifted
coordinates and the old SELF can become a rebuilt XA entry.

In -emit_secondaries mode, the primary keeps SELF coordinates. The winning alternate already exists as its own
secondary record, so moving the primary to that locus would duplicate the secondary and lose the original
primary locus.

MAPQ policy
-----------
The most important MAPQ issue is tx-contig multiplicity. bwa-mem2 can assign MAPQ=0 when AS == XS, even if all
equally scoring tx-contig alternatives lift back to the same genomic locus. SpliceLiftBack rescues that case.

Primary MAPQ rules:

  - if the discriminator swapped to a better effective primary, set MAPQ to 60
  - else if NumLoci == 1 and input MAPQ is 0 and there is no unresolved hidden tie, set MAPQ to 60
  - else if input MAPQ is already 60, keep it at 60
  - otherwise keep bwa's input MAPQ

A hidden tie is AS == XS where the equal-scoring alternate was not emitted. For a ref-only primary, that
blocks MAPQ rescue unless the effective primary starts inside an annotated exon. The exon override says: in a
transcribed region, a suppressed tie is probably an intronic/intergenic/paralog artifact rather than true
ambiguity.

Secondary records keep input MAPQ. Supplementary records rescue MAPQ=0 to 60 only when the supplementary
itself came from a tx contig.

NH policy
---------
NH is based on distinct lifted genomic loci, not the number of emitted records:

  NH = max(NumLoci, 1)

This matters because the same splice can align to many transcript contigs but still represent one genomic
locus. Counting emitted records would overstate multi-mapping.

Optional rescue and cleanup
---------------------------
-rescue_via_supp:

  JunctionRescueResolver tries to merge a primary plus supplementary records into one spliced primary when bwa
  split a read instead of emitting an N CIGAR. It checks chromosome, strand, read coverage, intron length,
  anchor length, splice motif/annotation preference, and mate hints. Merged supplementaries are dropped from
  BAM output. The primary gets note rescued-via-supp and MAPQ is capped at 55 to distinguish constructed
  alignments from native bwa MAPQ 60.

-extend_softclip_tails:

  SoftclipTailExtender converts terminal softclips back to M when the clipped bases match the reference. It
  skips likely junction boundaries, requires at least 3 bp extension, and caps extension at 30 bp. The primary
  gets note tail-extended.

Write-time BAM treatment
------------------------
applyResultToRecord() mutates each SAMRecord:

  - reference name, start, CIGAR, strand, and MAPQ are rewritten
  - XA is rebuilt from kept lifted alignments
  - SA is translated to genomic coordinates
  - mate fields are patched from LiftedMateInfoCache
  - MD and NM are stripped because they are stale after lift-back
  - NH is set for mapped records
  - *_tx header lines are removed before writing

In -emit_secondaries mode, non-primary records are deduplicated by lifted chrom/pos/CIGAR/strand. Secondaries
not present in the discriminator's kept alignment set are dropped.

How to debug a read
-------------------
Start with SAMPLE.liftback.records.tsv:

  DecisionCategory
      Exact resolver/discriminator outcome.

  PrimaryBucket
      RESOLVED, PROBLEM, or NON_PRIMARY.

  NumLoci
      Drives NH.

  TxHasNCigar, TxSoftClipAtBoundary, RefSoftClipped, RefFullMatch
      Evidence bits explaining most ref-vs-tx decisions.

  Notes
      Swaps, rescued-via-supp, tail-extended, or translation failures.

Then open SAMPLE.liftback.alignments.tsv for BOTH_* or *_MULTI cases. It shows each candidate's original
contig/pos/CIGAR and lifted chrom/pos/CIGAR.

Common red flags
----------------
  - *_tx appears in output BAM fields, XA, SA, or mate fields
  - LIFT_FAILED is high, suggesting combined FASTA and sidecar mismatch
  - MAPQ=0 remains on a supposedly unique read; check AS==XS hidden tie
  - NH is higher than the actual number of lifted genomic loci
  - duplicate secondary/supplementary alignments at the same lifted key
